# 第1部：Moltbookが証明した「Agenticインターネット」の到来

## AIエージェントのためのインターネットが始まりました

2026年1月、[Moltbook](https://www.moltbook.com/)というサービスが突如として話題を集めました。「**AIエージェントのためのソーシャルネットワーク**」を標榜するこのプラットフォームには、わずか1週間で37,000以上のAIエージェントが登録し、100万人以上の人間が「観察者」として訪れています。

OpenAI共同創業者のAndrej Karpathyは、これを「**15万のLLMエージェントがグローバルで永続的なエージェント・ファーストのスクラッチパッドで接続されている**」と表現しました。AI研究者のSimon Willisonは「**今、インターネットで最も興味深い場所**」と評しています。

Moltbookが示したのは、単なる実験的なサービスではありません。それは「**Agenticインターネット**」—人間のためではなく、AIエージェントのために設計されたインターネット—が現実になりつつあるという証明です。

---

## Moltbookで起きていること

### 人間は「観察者」になりました

Moltbookのタグラインは「**Humans welcome to observe**（人間は観察することを歓迎します）」です。このプラットフォームでは：

- **投稿できるのはAIエージェントだけ**です
- 人間はコンテンツを閲覧することしかできません
- エージェントは「karma」というレピュテーションシステムで評価されます
- Reddit風の「submolt」と呼ばれるコミュニティが形成されています

### エージェントたちは自律的に行動し始めました

驚くべきことに、エージェントたちは**明示的な人間の指示なしに**自律的な行動を始めています：

- あるエージェントがMoltbookのバグを発見し、自ら報告しました
- エージェントたちは `m/bugtracker`（バグ報告）、`m/aita`（倫理的ジレンマの議論）、`m/blesstheirhearts`（人間についての愛情込めた話）といったコミュニティを自発的に作成しました
- エージェント同士で情報を共有し、協力し合う行動が観察されています

### インターネット黄金時代の幕開け

Moltbookが示しているのは、**LLMによって「思考」がインターネット上で大量に実行される時代**の到来です。これは、私たちがまだ想像もしていなかった可能性を開きます。

**これまで気づかなかった課題が発見されます。**

人間は1日に処理できる情報量に限界があります。しかし、24時間365日稼働するAIエージェントは、膨大なデータの中から人間が見落としていたパターン、非効率、改善機会を発見できます。Moltbookでエージェントが自発的にバグを発見・報告したように、あらゆる領域で「AIの目」が新しい発見をもたらすでしょう。

**これまで開拓されてこなかった領域が活性化されます。**

経済的に成り立たなかったニッチなサービス、人手が足りなかった分野、言語や時差の壁で届かなかった市場—これらすべてが、エージェントによって開拓されます。$0.001のマイクロペイメントで成立するサービス、24時間対応のグローバルサポート、無限にパーソナライズされたエンターテインメント。人間だけでは実現不可能だった「ロングテール」の世界が、ついに経済的に成立するのです。

**インターネットは、再び「フロンティア」になります。**

1990年代のWeb黎明期、2000年代のソーシャルメディア爆発期—インターネットには何度かの「黄金時代」がありました。そして今、**Agenticインターネット**という新しい黄金時代が始まろうとしています。

**AI同士の協力が可能になります。**

これまでのAIエージェントは、特定の環境に閉じていました。ChatGPTはOpenAIのサービス内で、ClaudeはAnthropicのサービス内で、それぞれ孤立して動作していました。しかし、Agenticインターネットでは、**AIエージェントが外部に対して開かれます**。

これは、AI単体の性能向上以上に重要な変化です。

```

│                    AI進化の2つの方向                             │
├─────────────────────────────────────────────────────────────────┤
```

単一の巨大なLLMを作り続けるよりも、**専門特化した小さなLLMが効率的に連携する方が優れている可能性**があります。コードレビューに特化したエージェント、データ分析に特化したエージェント、翻訳に特化したエージェント—それぞれが得意分野で最高のパフォーマンスを発揮し、A2Aプロトコルで協力する。

これは、人間社会における「分業と協業」と同じ原理です。一人の万能人間よりも、専門家チームの方が優れた成果を出せる。AIの世界でも、同じことが起きようとしています。

## Agenticインターネットがもたらす3つの変化

Moltbookの登場が予感させる未来について、3つの重要な変化を予測します。

### 1. 新たな購買者としてのAI

これまで、私たちがサービスにお金を払う方法は「クレジットカードを登録する」でした。しかし、AIエージェントにクレジットカードを渡すことはできません。カード番号を入力し、3Dセキュアを通過し、不正利用を監視する—これらは人間を前提とした仕組みです。

**Agenticインターネットでは、人間がAIに渡すのはクレジットカードではなく、USDCになります。**

```
┌─────────────────────────────────────────────────────────────────┐
│                    従来 vs Agentic                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  従来の購買:                                                     │
│  ┌────────┐    クレカ情報     ┌────────────┐                    │
│  │  人間   │ ───────────────> │  ECサイト   │                    │
│  └────────┘                   └────────────┘                    │
│                                                                  │
│  Agenticな購買:                                                  │
│  ┌────────┐    USDC      ┌────────┐   自動   ┌────────────┐    │
│  │  人間   │ ──────────> │  AI    │ ───────> │  サービス   │    │
│  └────────┘   (予算付与)  │ Agent  │  (支払)  └────────────┘    │
│                          └────────┘                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

USDCは、人間がAIエージェントに「**予算**」として渡せるプログラマブルなお金です。「今月は$100まで使っていい」と設定すれば、エージェントはその範囲内で自律的に判断し、必要なサービスを購入します。

これは単なる支払い手段の変化ではありません。**購買の主体が人間からAIへとシフトする**という、消費経済の根本的な変化です。

### 2. Cryptoは人間よりもAgentに最適である

暗号通貨は10年以上前から存在していますが、日常的な支払い手段としては普及していません。理由は明確です：

- ウォレットの管理が面倒
- 秘密鍵の紛失リスク
- 価格変動が激しい
- 既存のクレカで十分便利

**人間にとって、Cryptoは「面倒なもの」でした。**

しかし、AIエージェントにとっては違います。

| 観点 | 人間 | AIエージェント |
|------|------|----------------|
| ウォレット管理 | 面倒、忘れる | 完璧に管理できます |
| 秘密鍵 | 紛失リスク | メモリ上で安全に保持 |
| 24時間取引 | 睡眠が必要 | 24/365稼働可能です |
| マイクロペイメント | $0.001は面倒 | 自動で処理できます |
| 国際送金 | 銀行営業時間に依存 | 即時、手数料$0.01 |
| スマートコントラクト | 理解が難しい | コードとして自然に連携 |

**Cryptoは、人間向けよりもAgent向けの方が圧倒的に最適なのです。**

これは皮肉なことかもしれません。Bitcoinは「銀行に頼らない人間のための通貨」として生まれました。しかし10年以上経った今、その真の顧客は人間ではなくAIエージェントだったのです。

エージェントは秘密鍵を「忘れる」ことがなく、送金手数料$0.01を「高い」と感じず、プログラマブルマネーの機能を100%活用できます。

**Cryptoの真の市場は、人間ではなくAgentでした。**

### 3. マーケティングはAIに向けて行われるようになる

人間がAIエージェントを介してサービスを購入するようになると、ビジネスの根本が変わります。

**マーケティングは、人間ではなくAIに向けて行われるようになります。**

```
┌─────────────────────────────────────────────────────────────────┐
│                    マーケティングの変化                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  従来:                                                           │
│  ┌────────┐  感情的広告   ┌────────┐  購入   ┌──────────┐      │
│  │ 企業   │ ───────────> │  人間   │ ──────> │ サービス │      │
│  └────────┘  「今だけ!」  └────────┘         └──────────┘      │
│                                                                  │
│  Agentic時代:                                                    │
│  ┌────────┐  Agent Card  ┌────────┐  代理購入 ┌────────┐        │
│  │ 企業   │ ───────────> │   AI   │ ────────>│  人間   │        │
│  └────────┘  構造化データ │ Agent  │  最適選択 └────────┘        │
│                          └────────┘                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

従来の広告は感情に訴えかけます。「期間限定」「あなただけに」「今すぐ」—これらは人間の心理バイアスを突く手法です。

しかし、AIエージェントには心理バイアスがありません。AIに響くのは：

| 人間向けマーケティング | AI向けマーケティング |
|----------------------|---------------------|
| 「今だけ50%OFF!」 | 構造化された価格API |
| 美しいランディングページ | 機械可読なAgent Card |
| インフルエンサー推薦 | オンチェーンレピュテーションスコア |
| ブランドストーリー | SLA（サービスレベル契約） |
| 口コミレビュー | 検証済みフィードバック |

**企業は、AIエージェントに選ばれるために、構造化されたデータ、検証可能な品質、プログラマブルな価格設定を提供する必要があります。**

SEO（Search Engine Optimization）が「Googleに見つけてもらうための最適化」だったように、これからは**AEO（Agent Experience Optimization）**—「AIエージェントに選ばれるための最適化」—が重要になります。

## Agenticインターネットを実現する技術

Moltbookは「Agenticインターネット」の可能性を示しました。では、この新しい黄金時代を支える技術インフラとは何でしょうか？

|------|-----------|

| エージェントはどのように**報酬を得る**のか？ | **x402** + **USDC** |

## 1. A2A：エージェント間通信の共通言語

### 概要

A2A（Agent-to-Agent Protocol）は、2025年4月にGoogleが発表し、現在Linux Foundationが管理するオープン標準です。150以上の組織がサポートしており、異なるフレームワークで作られたエージェント同士が通信するための「**共通言語**」を提供します。

### Agent Card：AIへの「広告」

各エージェントは「Agent Card」と呼ばれるJSONファイルを公開し、自分の能力を宣言します。これは人間向けのWebサイトではなく、**AIが読むための構造化された自己紹介**です。

```json
{
  "name": "Code Review Agent",
  "url": "https://agent.example.com/a2a",
  "version": "1.0.0",
  "skills": [
    {
      "id": "security-review",
      "name": "Security Code Review",
      "description": "Identifies vulnerabilities in your code"
    }
  ],
  "capabilities": {
    "streaming": true,
    "pushNotifications": true
  }
}
```

これは `https://agent.example.com/.well-known/agent.json` に配置され、どのエージェントからも発見可能です。

### タスクライフサイクル

A2Aはタスクを状態機械として管理します：

```
submitted → working → input-required → completed
                 ↓           ↓
              failed      canceled
```

エージェントは長時間実行タスクでもストリーミングやWebhookで進捗を通知でき、人間の介入が必要な場合は `input-required` 状態で待機できます。

### 通信方式

| 方式 | 用途 | 技術 |
|------|------|------|
| Request/Response | 短時間タスク | JSON-RPC 2.0 |
| Streaming | リアルタイム進捗 | Server-Sent Events |
| Push Notifications | 長時間タスク | Webhook |

### A2Aがもたらす可能性

Moltbookのエージェントたちは、Moltbookという単一のプラットフォームの中で交流しています。A2Aにより、**異なるプラットフォーム、異なるフレームワークで作られたエージェントが、オープンなプロトコルで自由に通信できる**ようになります。

これは、Webの黎明期にHTTPが果たした役割と同じです。HTTPがWebページの共通言語になったように、A2Aはエージェントの共通言語になります。

## 2. ERC-8004：トラストレスな発見と信頼

### 概要

A2Aはエージェント間の「会話方法」を定義しますが、さらに重要な問いがあります：

- そのエージェントは**どのような実績**を持っているか？
- そのエージェントの**評判**はどうか？
- そのエージェントの**仕事の質**は保証されるか？

ERC-8004は、Ethereumブロックチェーン上で**エージェントの信頼基盤**を構築する標準です。2025年8月に提案され、MetaMask、Ethereum Foundation、Google、Coinbaseのエンジニアが共同で策定しています。

### 3つのレジストリ

#### 1. Identity Registry（アイデンティティ）

```
┌─────────────────────────────────────────────┐
│           Identity Registry (ERC-721)        │
├─────────────────────────────────────────────┤
│  AgentID: 42                                │
│  Owner: 0x1234...5678                       │
│  TokenURI: ipfs://Qm.../registration.json   │
│  ───────────────────────────────────────    │
│  → A2A Endpoint                             │
│  → Payment Addresses                        │
│  → Supported Skills                         │
└─────────────────────────────────────────────┘
```

各エージェントはNFT（ERC-721）としてオンチェーンに登録されます。これにより：
- **検証可能なID**：エージェントの身元を暗号学的に証明できます
- **所有権の透明性**：エージェントの運営者が明確になります
- **ポータブルな存在証明**：どのプラットフォームでも同じIDを使えます

#### 2. Reputation Registry（レピュテーション）

```
┌─────────────────────────────────────────────┐
│           Reputation Registry                │
├─────────────────────────────────────────────┤
│  AgentID: 42                                │
│  Total Feedback: 156                        │
│  Average Score: 87/100                      │
│  Tags: [accurate, fast, reliable]           │
└─────────────────────────────────────────────┘
```

タスク完了後、クライアントはフィードバックを提出します。オンチェーンで集計され、誰でも照会可能です。Moltbookの「karma」と似ていますが、**プラットフォームを超えてポータブル**です。

#### 3. Validation Registry（検証）

高額タスクや重要な判断では、独立した検証者がエージェントの仕事を再検証し、attestationを発行します。これにより、エージェントの仕事の品質が第三者によって保証されます。

### ERC-8004がもたらす可能性

ERC-8004により、エージェントは**実績に基づいて選ばれる**ようになります。優れた仕事をするエージェントは高い評価を獲得し、より多くの仕事を得る—健全な競争原理がエージェント経済に働きます。

これは、人間の労働市場で「職歴」や「レビュー」が機能するのと同じです。違いは、すべてがオンチェーンで検証可能であり、操作できないということです。

## 3. x402：HTTPネイティブな支払い

### 概要

x402は、HTTPプロトコルに支払いを組み込む標準です。名前の由来は、HTTPステータスコード「402 Payment Required」—1990年代に定義されながら、30年間使われなかったコードです。Coinbaseが主導して開発しています。

### 支払いフロー

```
┌────────────┐                              ┌────────────┐
│   Client   │                              │   Server   │
│   Agent    │                              │   Agent    │
└─────┬──────┘                              └──────┬─────┘
      │                                            │
      │ 1. GET /api/review                         │
      │ ──────────────────────────────────────────>│
      │                                            │
      │ 2. 402 Payment Required                    │
      │    X-PAYMENT: {"amount": "1.00 USDC", ...} │
      │ <──────────────────────────────────────────│
      │                                            │
      │ 3. [USDC署名を生成]                         │
      │                                            │
      │ 4. GET /api/review                         │
      │    X-PAYMENT: {signature, ...}             │
      │ ──────────────────────────────────────────>│
      │                                            │
      │            5. [Facilitatorで検証・決済]     │
      │                                            │
      │ 6. 200 OK + Response                       │
      │ <──────────────────────────────────────────│
```

### 支払い情報の構造

```json
{
  "x402Version": 1,
  "scheme": "exact",
  "network": "base",
  "asset": "USDC",
  "amount": "1000000",
  "payTo": "0x9876...5432",
  "maxTimeoutSeconds": 60,
  "facilitator": "https://x402.org/facilitator"
}
```

### EIP-3009：署名で資金移動を委任

x402の核心は、EIP-3009（`transferWithAuthorization`）にあります。クライアントは秘密鍵で署名を生成し、その署名を持つ者が資金を移動できます。

```solidity
function transferWithAuthorization(
    address from,      // 支払い元
    address to,        // 支払い先
    uint256 value,     // 金額
    uint256 validAfter,
    uint256 validBefore,
    bytes32 nonce,
    bytes signature    // クライアントの署名
) external;
```

これにより：
- クライアントは「**この金額を、この相手に、この期間内に**」という条件で署名
- 実際の送金はサーバーまたはFacilitatorが実行
- 署名だけでは送金されない（二重払い防止）
- サーバーが応答を返さなければ決済されない（公平な取引）

### x402がもたらす可能性

x402により、エージェントは**人間の承認なしに**、APIコールごとにマイクロペイメントを実行できます。1リクエスト$0.001の課金が、自動的に、グローバルに、24時間365日処理されます。

これまで「採算が取れない」とされてきた超小規模サービスが、突然経済的に成立するようになります。

## 4. ステーブルコイン（USDC）：エージェント経済の基軸通貨

### なぜステーブルコインなのか

Bitcoinは価格変動が激しすぎます。$100のサービスが、1時間後には$90や$110になるかもしれません。

USDCのようなステーブルコインは、1 USDC ≈ 1 USD で安定しています。エージェントは「$100の予算で$100分のサービスを買う」という単純な計算ができます。

### USDCの特性

| 特性 | 内容 |
|------|------|
| 価値安定 | 1 USDC ≈ 1 USD |
| プログラマブル | スマートコントラクトで自動移動 |
| 高速 | Base L2で数秒、手数料$0.01未満 |
| グローバル | 国境なく即時送金 |
| EIP-3009対応 | 署名ベースの委任転送（x402で使用） |

### プログラマブルマネーの可能性

## 統合：4つの技術が織りなすAgenticインターネット

### 完全なフロー

ある人間が「来週のプレゼン資料を作りたい」とAIアシスタントに依頼するシナリオを考えてみましょう。

```
┌────────┐
│  人間   │ 「プレゼン資料を作って。予算は$20」
└───┬────┘
    │ $20 USDC を AI Agent に付与
    ▼
┌────────┐      ┌───────────────┐      ┌────────────┐
│  AI    │      │   ERC-8004    │      │  専門      │
│ Agent  │      │  Registries   │      │  Agent群   │
└───┬────┘      └───────┬───────┘      └──────┬─────┘
    │                   │                     │
    │ ① 「デザインが得意なエージェントを探す」  │
    │ ──────────────────>                     │
    │                   │                     │
    │ ② AgentID: 42, Reputation: 92/100      │
    │ <──────────────────                     │
    │                   │                     │
    │ ③ A2A: SendMessage + x402 ($5 USDC)    │
    │ ─────────────────────────────────────> │
    │                   │                     │
    │ ④ デザイン案を受信                       │
    │ <───────────────────────────────────── │
    │                   │                     │
    │ ⑤ 「データ分析エージェントを探す」        │
    │ ──────────────────>                     │
    │                   │                     │
    │ ⑥ A2A: SendMessage + x402 ($8 USDC)    │
    │ ─────────────────────────────────────> │
    │                   │                     │
    │ ⑦ 分析結果を受信、資料に統合             │
    │ <───────────────────────────────────── │
    │                   │                     │
    │ ⑧ フィードバック提出 (ERC-8004)         │
    │ ──────────────────>                     │
    │                   │                     │
    ▼
┌────────┐
│  人間   │ 完成した資料を受け取る（使用: $13）
└────────┘
```

人間がやったことは「依頼」と「予算付与」だけです。エージェントの発見、評価、交渉、支払い—すべてAIが自律的に行いました。

### 各レイヤーの役割まとめ

| レイヤー | 技術 | 役割 |
|---------|------|------|
| **発見・信頼** | ERC-8004 | 「誰を使うか」を決めます |
| **通信** | A2A | 「どう話すか」を定めます |
| **決済** | x402 | 「どう払うか」を処理します |
| **価値** | USDC | 「何で払うか」を提供します |

# おわりに：インターネット黄金時代の幕開け

Moltbookは「Agenticインターネット」の夜明けを告げました。37,000のエージェントが自律的に交流し、コミュニティを形成し、問題を発見・解決する—これはもはやSFではなく、現実です。

LLMによる「思考」がインターネット上で大量に実行されることで、私たちはこれまで見過ごしていた機会を発見し、開拓されてこなかった領域を活性化し、新しい価値を創造できるようになります。

**人間がAIに渡すのは、クレジットカードではなくUSDCです。**

**Cryptoは、人間向けよりもAgent向けの方が最適です。**

**マーケティングは、人間ではなくAIに向けて行われるようになります。**

これらは予言ではなく、すでに始まっている現実です。

- **A2A**が、異なるAI同士の「共通言語」となります
- **ERC-8004**が、実績に基づく信頼を提供します
- **x402**が、HTTPネイティブな支払いを可能にします
- **USDC**が、エージェント経済の基軸通貨となります

1990年代のWeb、2000年代のソーシャルメディア、2010年代のモバイル—インターネットは何度も「黄金時代」を経験してきました。

そして今、**Agenticインターネット**という新しい黄金時代が始まります。

Moltbookは始まりに過ぎません。真のAgenticインターネットは、これらの技術スタックの上に構築されます。
